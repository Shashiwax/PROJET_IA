# Environment Setup Instructions

## 1. Prerequisites

This project is designed to run inside **WSL2 (Ubuntu)** on Windows.

To verify your WSL version:

```bash
wsl -l -v
````

Expected:

```
Ubuntu    Running    2
```

Make sure the repository is cloned **inside WSL**, not in a Windows path (`/mnt/c/...`).

---

## 2. Install system dependencies inside WSL

Update the system:

```bash
sudo apt update && sudo apt upgrade -y
```

Install required tools:

```bash
sudo apt install -y build-essential curl wget git ngspice
```

---

## 3. Install uv (Python toolchain manager)

Install via curl:

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

Load uv into the current shell:

```bash
source ~/.local/bin/env
```

(Optional) Make uv available automatically in future sessions:

```bash
echo 'source ~/.local/bin/env' >> ~/.bashrc
```

Verify installation:

```bash
uv --version
```

---

## 4. Git + SSH Setup

### 4.1 Configure Git identity

```bash
git config --global user.name "Your Name"
git config --global user.email "your_email@example.com"
```

### 4.2 Generate an SSH key

```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
```

Start the SSH agent and add the key:

```bash
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
```

Display the public key:

```bash
cat ~/.ssh/id_ed25519.pub
```

Copy this key to GitHub:
**GitHub → Settings → SSH and GPG Keys → New SSH Key**

---

## 4.3 Clone the repository (inside WSL)

```bash
git clone git@github.com:<username>/<repository>.git
cd <repository>
```

---

## 5. Create the Python environment with uv

Run the following inside the project folder:

```bash
uv sync
```

This command will:

* Create a `.venv/` directory
* Install the appropriate Python version
* Install all dependencies from `pyproject.toml` and `uv.lock`

---

## 6. Running the project

Use `uv run` to execute Python inside the virtual environment:

```bash
uv run python main.py
```

Start an interactive Python session:

```bash
uv run python
```

Enter a shell inside the environment:

```bash
uv run --shell
```

---

## 7. Verify ngspice integration (optional)

Check the ngspice version:

```bash
ngspice -v
```

Test Python + ngspice:

```bash
uv run python -c "import subprocess; print(subprocess.run(['ngspice','-v'], capture_output=True).stdout.decode())"
```

---

## 8. Project structure

A typical project layout:

```
project/
│── README.md
│── pyproject.toml
│── uv.lock
│── src/
│     └── main.py
│── netlists/
│     └── rc.cir
│── .venv/  (auto-generated by uv)
```

---

## 9. Adding new dependencies

To install Python packages into the uv environment:

```bash
uv add <package-name>
```

Example:

```bash
uv add numpy pandas
```

---

## 10. Notes on storage and performance

* Keep the project **inside WSL**, not in Windows directories (`/mnt/c/...`).
* WSL must stay installed on the **internal SSD (C:)** for optimal performance.
* Large datasets may be stored on an external HDD or SSD and accessed via `/mnt/<drive>/`.
* Do **not** install Python environments on HDDs (slow I/O).
